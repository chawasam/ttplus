🛡️ Guard / Block Action
S Phase 1
เพิ่ม action ที่ 5 ให้ผู้เล่นเลือก "ตั้งรับ" แทน "โจมตี" — ลด damage รับ 40–50% + รับ MP เล็กน้อย
⚙️ เพิ่ม action type guard ใน processAction() — ตั้ง flag isGuarding:true บน playerState
⚙️ ตอนคำนวณ enemy attack: ถ้า isGuarding ลด damage 45%, +5 MP restore, reset flag
🎮 Frontend: button "🛡️ ตั้งรับ" ใน battle action bar, แสดง "GUARDED! -45%" บน damage log
💡 Warrior passive: Guard ลด damage 60% (แทน 45%) เพื่อทำให้ class ต่างกัน

💥 Crit System Upgrade
S Phase 1
เพิ่ม crit multiplier ที่ปรับได้ (ตอนนี้ hardcode ×1.5), เพิ่ม "Super Crit" mechanic และ crit impact บน frontend
⚙️ เพิ่ม critMult field บน playerState (default 1.5, เพิ่มจาก gear/buffs)
⚙️ "Super Crit" — ถ้า roll ≤ critChance/3: ×2.2 damage + log ต่างจาก normal crit
⚙️ Assassin passive: critMult 1.9 (แทน 1.5), Soulseer: เพิ่ม MP ทุก crit
🎮 Frontend: "CRITICAL!!" สีแดง size ใหญ่, "SUPER CRIT!!" pulse animation
// combat.js — crit calculation upgrade const critRoll = Math.random() * 100; let isCrit = false, isSuperCrit = false; let critMult = playerState.critMult || 1.5; if (critRoll <= playerState.critChance / 3) { isSuperCrit = true; critMult = 2.2; } else if (critRoll <= playerState.critChance) { isCrit = true; } const finalDmg = Math.floor(baseDmg * (isCrit || isSuperCrit ? critMult : 1)); // Soulseer passive: +8 MP on crit if ((isCrit || isSuperCrit) && charClass === 'soulseer') { playerState.mp = Math.min(playerState.mp + 8, playerState.mpMax); }
🐉 Named Enemy Moves
M Phase 1
ศัตรูมี "ชื่อ skill" แทนแค่ "โจมตี" — บาง move มีผลพิเศษ เพิ่ม personality และ readability
📁 เพิ่ม moves[] array ใน monsters.js แต่ละ move มี: name, emoji, msgTemplate, dmgMult, effect?, telegraphed?
⚙️ combat.js: ก่อน enemy turn สุ่ม move จาก monster.moves — ถ้า move มี telegraphed:true broadcast "⚠️ [name] กำลังจะ [move]!" round ก่อน
💡 Telegraphed move: Guard ใน round นั้น ลด damage เพิ่มพิเศษ 70% (สร้าง read-and-react gameplay)
// monsters.js — example named moves { monsterId: 'giant_spider', moves: [ { name: 'ฉีกด้วยเขี้ยว', emoji: '🕷️', dmgMult: 1.0, weight: 50 }, { name: 'Web Wrap', emoji: '🕸️', dmgMult: 0.6, weight: 30, effect: { type: 'SLOW', duration: 2 } }, { name: 'Venom Strike', emoji: '☠️', dmgMult: 0.8, weight: 20, telegraphed: true, effect: { type: 'POISON', duration: 3, dmgPerTurn: 8 } }, ] }, // combat.js — weighted random move selector function pickMove(monster) { if (!monster.moves?.length) return null; const total = monster.moves.reduce((s, m) => s + (m.weight || 1), 0); let r = Math.random() * total; for (const m of monster.moves) { r -= m.weight || 1; if (r <= 0) return m; } return monster.moves[0]; }

🌀 Status Effects ที่มีความต่างกันชัด
S Phase 1
ปัจจุบัน BURN/BLEED/POISON overlap กัน — ทำให้แต่ละ status ต่างกันจริงๆ
🔥 BURN — DoT ทุก turn, ลด DEF 10%, ถ้า apply ซ้ำ: extend duration
🩸 BLEED — DoT เพิ่มขึ้นทุก turn (1→2→3→...), หายเมื่อ heal
☠️ POISON — DoT คงที่ แต่ลด healing ที่รับ 30%
⚡ STUN — skip turn + เพิ่ม damage รับ 15% (ต่างจาก SLOW)
🐢 SLOW — ATK -25%, ไม่ skip turn แต่ลด speed สำหรับอนาคต
🎯 MARKED — เพิ่ม crit chance ของ attacker ต่อ target นี้ +15% (ขยาย mechanic เดิม)
👁️ CURSE — ลด stat random 1 ค่า (ลบ ATK หรือ DEF หรือ SPD 20%)

⚡ Battle Momentum System
M Phase 2
Resource ที่ build จากการโจมตี/ถูกโจมตี — ใช้ unlock special actions ชั่วคราว สร้างความตึงเครียด

Major Class Mechanics — แต่ละ class เล่นต่างกันจริงๆ

Unique resource และ mechanic ต่อ class — เปลี่ยน experience การเล่น

🪓 Berserker — Rage Stacks
แทน MP ธรรมดา Berserker สะสม Rage จาก damage รับ
⚙️ Rage resource (0–10 stacks) แทน MP
⚙️ รับ damage → +1 Rage, ทุก Rage stack → +5% ATK
⚙️ Skills ใช้ Rage แทน MP: Rage Smash (3 stacks), Blood Frenzy (5 stacks)
⚙️ 10 stacks = FRENZY: attack 2× ครั้ง/turn เป็น 2 turns, reset Rage
💡 Low HP → more damage → more rage → stronger — comeback mechanic
☠️ Necromancer — Minion System
kill ศัตรู → soul shard → summon minion ที่โจมตีทุก turn
⚙️ Kill monster → +1 Soul Shard (max 3)
⚙️ Skill "Raise Dead" (2 shards): summon skeleton, โจมตี 5–10 damage ทุก turn
⚙️ minion[] array บน battleState, process หลัง player turn
⚙️ Minion อยู่ได้ 5 turns หรือจน battle จบ
💡 สร้าง board state — ผู้เล่นจัดการ resource ข้ามเทิร์น
⚙️ Engineer — Turret Deploy
วาง Turret ที่ยิงซ้ำทุก round — ต้องจัดการ MP vs ผลตอบแทน
⚙️ Skill "Deploy Turret" (15 MP): วาง turret ชาร์จ 1 turn
⚙️ Turret ยิงอัตโนมัติ: 8–15 damage ทุก turn, lasts 4 turns
⚙️ max 1 turret ต่อ battle — upgrade ด้วย "Overclock" (10 MP: +50% dmg, -2 turns)
⚙️ turretState บน battleState: { active, turnsLeft, dmg }
⚡ Rifter — Void Charges
สะสม Void Charge จาก special actions → release เป็น burst damage
⚙️ Void Charge (0–5) สะสมจาก certain skills
⚙️ Skill "Void Pulse" (ไม่เสีย MP): สะสม +1 charge, 70% damage ปกติ
⚙️ Skill "Rift Collapse" (20 MP): release ทุก charge × 25 damage → reset
⚙️ 5 charges: passive +25% all damage ชั่วคราว
💡 Wind-up → payoff pattern — patience-reward gameplay
🎵 Bard — Song Stacking
เล่น Song ที่ stack effect ขึ้นทุก turn — ยิ่งอยู่นาน ยิ่งแรง
⚙️ Active song: "Battle Hymn" (8 MP) — บน turn เริ่มต้น, ทุก turn: +3% ATK stack
⚙️ Max stack = 10 turns (30% bonus ATK) — หายเมื่อ switch song
⚙️ Song 2: "Mending Melody" — heal 5% HP/turn
⚙️ Song 3: "Requiem" — ลด enemy ATK 15%, stack ทุก turn
💡 ต้องเลือก song เร็ว และ commit — strategic pacing
👻 Phantom — Phase System
สลับระหว่าง Material/Ethereal plane — ต่างกัน damage รับ/ให้
⚙️ State: material (default) vs ethereal
⚙️ Skill "Phase Shift" (10 MP): สลับ state เป็น ethereal 2 turns
⚙️ Ethereal: damage รับ -50%, แต่ physical attack ให้ damage -30%
⚙️ Skill "Soul Strike" (ethereal only): magic damage ×1.8 — ignore DEF
💡 Timing เป็นกุญแจ — phase in/out ตาม enemy pattern

💡 Notes & Design Principles
✅ หลักการออกแบบ
1️⃣ Additive — ทุก feature เพิ่มบน existing code, ไม่ rewrite
2️⃣ Backward compatible — battle state เก่า load ได้ (field ใหม่ optional)
3️⃣ Class identity — Phase 3 ทำให้ class เลือก playstyle ต่างกันจริงๆ
4️⃣ Read-and-react — telegraphed moves, shield phase สร้าง skill expression
5️⃣ Visual feedback — ทุก mechanic ต้องมี log + frontend indicator
⚠️ ระวัง
🔴 Firestore read/write ต่อ turn แพง — batch update ใน 1 write เสมอ
🔴 battle state grow ใหญ่ขึ้น — cleanup stale battles ยังสำคัญ
🔴 Class-specific state (rageStacks) ต้อง initialize ตาม charClass ใน startBattle
🔴 Frontend ต้องแสดง resource ใหม่ (momentum bar, rage stacks) ไม่งั้น player ไม่รู้
🟡 Test balance — Limit Break ×3 อาจแรงเกินไปสำหรับ early game
